# Lovable Landing Page Prompt

Use the following prompt in Lovable:

```text
Create a polished, modern, professional landing page for a product called "Asset Resolve" - a role-based asset lifecycle management platform built for organizations that need more than a simple CRUD inventory tool.

Product positioning:
Asset Resolve is a full-stack asset management platform that covers asset inventory, assignments, borrow requests, verification campaigns, discrepancies, maintenance, disposal, notifications, reporting, admin governance, and backend-enforced RBAC. It should feel like a serious enterprise operations product and also work well as a capstone / portfolio showcase.

Target audience:
- recruiters and hiring managers evaluating engineering depth
- lecturers and project reviewers
- operations teams, asset officers, managers, and administrators

Core story:
This is not just an inventory dashboard. It is an asset operations platform where each role sees the workflows they are responsible for, while the backend remains the source of truth for permissions and access scope.

Visual direction:
- modern SaaS quality with a clean enterprise feel
- sharp layout, strong spacing, premium cards, subtle motion
- professional color direction with deep slate / navy foundations, crisp accent color, and clean neutrals
- avoid generic purple startup aesthetics
- use tasteful gradients, layered backgrounds, and dashboard-style visual framing
- responsive and accessible on desktop and mobile

Important: generate or suggest suitable visuals/images for the page.
Include product-relevant imagery such as:
- hero illustration or mockup showing an asset dashboard and workflow cards
- dashboard-preview style visuals for role-based modules
- workflow visuals for borrow approvals, verification, maintenance, and disposal
- security / RBAC themed illustrations or abstract trust visuals
- clean product UI mockups that match the tone of an enterprise operations platform

Page structure:
1. Hero section
2. Trust/value strip
3. Problem statement
4. Feature highlights
5. Role-based views section
6. Workflow overview section
7. Security and RBAC section
8. Product preview / screenshots section
9. Architecture and reliability highlights
10. Demo / Try Now CTA section
11. FAQ
12. Footer

Section guidance:

Hero:
- headline should emphasize full asset lifecycle visibility and role-aware operations
- include a short subheadline explaining that the platform covers inventory, borrowing, verification, discrepancies, maintenance, and disposal
- primary CTA button: "Try Now"
- secondary CTA button: "View Demo"
- include GitHub and Documentation buttons in the hero or nearby
- show a premium dashboard mockup or generated product hero visual

Trust/value strip:
- concise proof points such as RBAC, workflow automation, verification tracking, audit visibility, CI-tested delivery

Problem section:
- explain the pain of stale asset records, fragmented approvals, spreadsheet-based tracking, and inconsistent access control

Feature highlights:
- asset inventory and ownership tracking
- borrow request approvals
- verification campaigns and discrepancies
- maintenance scheduling
- disposal review
- notifications and reporting
- global search

Role-based views section:
- show cards or tabs for Admin, Officer, Manager, Employee, Technician, Auditor
- emphasize that each role sees only the workflows and data they are allowed to access

Workflow overview section:
- visually explain how assets move through assignment, borrowing, verification, maintenance, and disposal
- use a clean diagram-like presentation with cards, arrows, or timeline blocks

Security / RBAC section:
- emphasize backend-enforced RBAC, scoped visibility, and audit-friendly operations
- this should feel like a serious differentiator, not a generic bullet

Product preview section:
- include product-style screenshots or generated UI visuals
- show dashboard cards, tables, workflow statuses, and search results

Architecture / reliability section:
- mention Spring Boot, PostgreSQL, React, Vite, Flyway, CI, and automated tests
- position these as trust signals for quality and delivery maturity

CTA section:
- prominent "Try Now" button
- also include "Live Demo", "GitHub", and "Documentation" links
- use placeholder links:
  - Try Now -> https://your-deployed-app-url.example
  - Live Demo -> https://your-demo-url.example
  - GitHub -> https://github.com/your-org/asset_managemt_final
  - Documentation -> https://your-docs-url.example

FAQ suggestions:
- Is this only an inventory app?
- How does role-based access work?
- Can it support verification and maintenance workflows?
- What stack is it built with?
- Is there a demo or documentation available?

Design requirements:
- modern, premium, recruiter-friendly, and reviewer-friendly
- clean card/grid layout
- section-by-section hierarchy must be obvious
- subtle scroll or reveal animations
- accessible typography and color contrast
- strong CTA visibility
- write strong, specific messaging based on the actual product
- avoid vague generic SaaS filler copy

Output:
- produce the complete landing page
- include section-by-section content
- include the visuals or image prompts/suggestions inline where appropriate
- make it feel like a serious full platform, not a toy CRUD admin panel
```
